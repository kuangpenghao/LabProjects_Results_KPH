# 第008主版本：全局结构风险与结构邻域训练

## 结论

本主版本合并全局结构风险重排、结构邻域 contrastive loss、以及当前 top-k 可靠竞争项修复。结论是：前两条方案都能执行，但尚未形成稳定收益；当前活跃工作是修复结构邻域 loss 的竞争项选择，让它只关注更可靠、更强的局部结构竞争。

## 已完成证据

- 全局结构风险重排入口可运行，但第一版恢复结果基本复刻旧 选择器，没有带来新的选择能力。
- 结构邻域 loss 默认关闭，代码可编译，可产生非空结构诊断，并完成 tiny Berkeley+BERT save/load/test。
- PTB256/MCTB64 BERT-base 方向性诊断显示：PTB test 基本中性略负，MCTB macro delta 为 -0.43；dialogue、literature、review 回退，forum、law 小涨。
- 因此，原始结构邻域 loss 被降级，不值得为同一目标继续恢复 BERT-large cache。
- 当前 top-k 可靠竞争项修复已经通过语法检查和功能 smoke：默认行为保持不变，显式打开新参数后按每个 gold span 筛正向竞争项并取 top-k。

## 当前活跃判断

原始 loss 的问题可能不是“结构信号完全无效”，而是训练时把过宽、过弱、甚至方向不清的竞争项混在一起。top-k 修复试图减少噪声，让 loss 更接近“强竞争结构之间的可学习排序”。

## 成功标准

只有在同一训练/评测预算下，top-k 修复至少做到 PTB 不显著伤害、MCTB 多数域不退化，才值得进入更大规模诊断。若仍然 macro 负向，则结构邻域训练线应继续降级，转向新的候选生成或 reranking 机制。

## 下一步

继续当前 top-k 可靠竞争项实验，先做小规模 matched diagnostic，再决定是否扩大。报告更新应继续写在本主版本内，不再为每个 smoke 新开版本。

## 完整阶段记录

本节保留本主版本内的连续实现、验证、问题定位和判断依据。后续同一方向的补充验证继续写入本主版本，只有研究问题、候选机制、比较合同或结论发生实质变化时才新开主版本。

### 阶段 1：全局结构风险重排路线选择

#### 结论

第 025 轮已经证明：继续调局部 selector 阈值不值得。下一步选择 **完整树级全局结构风险重排**，而不是立刻进入更重的训练侧一致性或从头实现 true n-best / forest reranking。

这条路线的核心判断是：候选池里确实有好树，但局部 span margin 不足以判断该不该换。下一轮要把选择依据升到整棵树层面，显式考虑结构距离、规则签名变化、标签方向、空结构倾向和模型分数兼容性。

#### 依据

最新放大验证给了一个很清楚的分叉：

- max-empty selector 在 MCTB 五域各 1000 句上宏平均只剩 `+0.0048`。
- literature 域转负：`-0.0241`。
- 整体只有 `21` 胜、`19` 负、`4960` 平。
- 同一候选池的 oracle / safe-oracle 仍有明显空间，五域 safe-oracle delta 约为 `+0.23` 到 `+0.69`。

这说明问题不是“有没有候选”，而是“如何无标签地挑候选”。继续在 `target_margin`、`target_empty_margin`、`span_len` 上找更细阈值，预期只会复现 `21/19` 这种几乎抵消的结果。

#### 三个候选方向比较

| 方向 | 判断 | 原因 |
| --- | --- | --- |
| 完整树级全局结构风险重排 | 选择 | 直接利用现有候选池，改的是选择原则；成本低，能最快验证是否可以从 oracle 空间中稳定取益。 |
| true n-best / forest reranking | 暂缓 | 机制更完整，但需要更深解码接口；应先确认现有完整树候选能否被全局风险特征稳定选择。 |
| 训练侧结构一致性 | 暂缓 | 可能更有上限，但训练成本和混杂更大；应在测试时选择机制继续失败后再升级。 |

#### 选中路线的机制

下一轮不再只看单个 span 的局部 margin。候选树会先通过现有完整树候选生成入口产生，然后用一个默认关闭的全局重排器打分。

第一版应优先加入这些无标签特征：

1. 结构距离：边界变化、标签签名变化、单标签替换还是多标签扰动。
2. 规则风险：父子规则签名变化、组合规则是否稀有或异常。
3. 空结构风险：继续保留 max-empty 信号，但把它作为风险特征而不是硬阈值终点。
4. 分数兼容性：候选树相对原树的模型分数差，而不是只看触发 span 的局部分数差。
5. 标签方向风险：例如 `S -> VP`、`VP -> S+VP` 这类方向在 PTB dev 上的胜负统计是否稳定。

训练和调参边界保持不变：只允许用 PTB dev 做选择规则开发和安全门控；MCTB 标签只用于固定方案评分，不用于筛选、调参或早停。

#### 成功标准

第一阶段只要求通过诊断门：

- PTB dev 不增加边界 / split churn，labeled-F1 非负。
- MCTB 128x5 或 1000x5 固定评分不再是 `21/19` 式抵消，至少要净胜明显大于净负。
- 五域不能出现像 literature `-0.0241` 这种明确负向域。
- 若收益仍小，也必须解释为“覆盖不足”或“方向错误”，不能只继续调阈值。

#### 降级标准

如果全局结构风险重排仍不能从现有候选池中稳定选出好树，则当前完整树候选池路线应继续降级。届时优先转向 true n-best / forest reranking 或训练侧结构一致性，而不是再回到局部阈值微调。

#### 下一步

1. 在现有完整树候选探针中补默认关闭的全局重排器和特征导出。
2. 先跑 PTB dev 小验证，确认 feature-off 不变且全局重排不制造结构风险。
3. 通过后再用固定 MCTB 设置验证，不用目标域标签做任何选择。

### 阶段 2：全局结构风险重排器实现与安全烟测

#### 结论先行

本轮完成了默认关闭的完整树级全局结构风险重排入口，但还没有把它晋级为有效性能方案。

第一版真实 PTB8 烟测暴露出一个关键风险：没有总风险预算时，重排器会误选一个完整树候选，导致 labeled F1 下降 `-0.7782` 点。加入 `max-score=1.0` 的保守保护后，同样 PTB8 检查回到 `0.0000`，且 feature-off / identity gate 的签名和树字符串不变数量均为 `0`。

放大验证已经完成：加短跨度保护后，PTB128 从 `-0.0556` 拉回到 `+0.0556`，退化样例从 `2` 个降到 `0`。但这个结果与旧 选择器完全持平，没有证明全局风险重排器带来额外选择能力；因此本轮结论是安全保护有效，但方案不晋级到 MCTB。

#### 本轮目标

第025轮 MCTB 1000x5 放大验证证明 max-empty 选择器阈值线基本失效：五域宏平均只剩 `+0.0048`，literature 域转负 `-0.0241`，整体只有 `21/19/4960` 个胜/负/平事件。因此第026轮选择从局部阈值转向完整树级风险选择。

本轮具体目标是：

1. 在现有完整树候选池上接入默认关闭的全局风险重排器。
2. 不使用 MCTB 标签，不改变默认 Berkeley+BERT 解析路径。
3. 先用最小真实解析烟测检查是否存在明显 PTB 安全问题。
4. 如果小烟测暴露问题，先修保护条件，而不是直接放大到 MCTB。

#### 依赖证据

- 第025轮：选择器阈值线在 MCTB 1000x5 上几乎归零，说明继续小阈值调参不值得。
- 第026轮：在 true n-best / forest、训练侧结构一致性、全局风险选择器中，选择最便宜且最贴近现有候选池的完整树级全局结构风险重排。
- 现有完整树候选探针：已经能生成控制树、候选树、oracle / safe-oracle / selector 汇总，适合作为低风险接入口。
- 已确认 baseline：Berkeley+BERT 比较合同保持不变，本轮不重训、不替换 baseline。

#### 实现内容

本轮只修改候选探针，不改训练主链路：

- 增加完整树规则签名特征，用父子标签关系刻画候选树相对控制树的结构变化。
- 在候选关系中加入 rule-signature added / removed / symmetric-diff。
- 增加默认关闭的 global-risk selector：
  - target-label margin
  - empty-structure margin
  - boundary symmetric diff
  - label diff
  - rule-signature diff
  - label-direction bias
  - candidate rank
- 增加保护门：
  - boundary / label / rule diff 上限
  - margin 与 empty-margin 约束
  - span 长度约束
  - `global-risk-max-score` 总风险预算，默认 `1.0`
- 汇总报告支持在启用全局 selector 时输出 `global selector delta`，默认关闭时不影响既有输出格式。

#### 验证结果

##### 代码级检查

- 本地 `py_compile` 通过。
- 本地 import 级小测被环境缺少 `numpy` 阻塞，因此未把它当成代码失败。
- 在 parser runtime 环境中重复 import 级 selector 小测，输出 `global-risk selector smoke ok 0.8125`，说明函数入口可加载、可打分、可选择。

#### 当前判断

本轮不是一次成功的性能验证，而是一次必要的机制保护与降级记录。

- 已完成：默认关闭的完整树全局风险重排入口。
- 已完成：feature-off / identity gate 保持不变。
- 已发现并修复：无总风险预算会导致 PTB 小样本误选。
- 已完成：PTB128 放大验证确认短跨度保护能消除退化。
- 未成立：global 选择器没有超过旧 选择器，不能作为性能路线晋级。
- 未执行：任何 MCTB 评分或跨域收益声明，因为 PTB dev 选择能力不足。

#### 下一步

1. 停止围绕当前 global-risk 手工特征继续抠阈值。
2. 保留完整树候选池与风险特征代码，作为后续候选分析和诊断工具。
3. 下一步转向更底层的选择机制：优先比较 true n-best / forest reranking 与训练侧结构一致性，选择一个能真正利用候选 oracle 空间、而不是复刻旧保守 选择器的方案。

### 阶段 3：结构邻域对比损失 smoke

#### 状态

默认关闭实现、一棵树 smoke、tiny Berkeley+BERT 保存/加载/测试 smoke，以及一个有边界的 PTB/MCTB 匹配诊断都已完成。机制本身能触发，但当前形式不值得晋级：方向性 BERT-base 诊断在 PTB test 基本持平，在 MCTB macro 上为负。

#### 目标

检验 PTB-only 的训练侧结构邻域一致性目标，是否能在不改变默认 Berkeley+BERT 路径的前提下产生有限、可解释的诊断信号，并据此判断是否值得进入更强的 PTB/MCTB 匹配对照。第一道门槛是一棵树有限损失和 tiny BERT 执行；有边界诊断使用相同 PTB 训练数据对比 control 与 mechanism，MCTB 只作为评测集。

#### 执行结果

在 Berkeley parser 训练 loss 中实现了默认关闭的结构邻域 loss hook。这个 hook 从 PTB 金标树邻域构造局部对比项，而不是继续使用宽泛的 quota-based 强负例：

- 同跨度错误标签替代项；
- 邻近重叠 span；
- 父子重叠约束；
- 兄弟节点合并约束；
- 兄弟非空约束。

最终的一棵树 seed smoke 对 feature-off 与 feature-on 使用相同随机初始化，在远端 parser 环境中强制 CPU，并写出干净 JSON。对一棵 49-token PTB 树，它得到有限 loss、`244` 个正向结构邻域项，在 `structure_neighborhood_loss_scale=0.1` 下 feature-on minus feature-off loss delta 为 `+0.167572`。关系计数为 `nearby_overlap=111`、`parent_child_overlap=58`、`same_span_wrong_label=31`、`sibling_merge=18`、`sibling_nonempty=26`。

输出保存在 `experiments/main/span_boundary_calibration/structure_neighborhood_loss_smoke_20260429_0831/`。

随后 tiny Berkeley+BERT 路径 smoke 在同一个 tiny PTB 子集上训练两个匹配条件：control 关闭新 loss，mechanism 打开 `structure_neighborhood_loss_scale=0.02`、`structure_neighborhood_loss_margin=1.0` 和归一化结构 loss。第一次远端尝试完成了 train/save/load/test，但脚本用来检查结构项是否出现在训练日志里的 assertion 失败。定位结果是 parser 统计字段叫 `total_terms` / `positive_terms`，而训练日志 wrapper 最初读取的是较早的 `term_count` / `positive_term_count`。这是日志/断言不一致，不是解码崩溃证据。

修复 stats 读取后，同一个 tiny Berkeley+BERT smoke 成功完成。mechanism 训练日志在两棵训练树上分别输出 `244` 和 `23` 个 structure-neighborhood terms，control 日志没有这些项。两个条件都完成训练、checkpoint 保存、reload 和 test。tiny dev FScore 两边都是 `1.98`，一棵树 test FScore 两边都是 `nan`，因为 test sample 退化；这些数字只作为链路闭环证据保留，不作为性能结果。

成功输出保存在 `experiments/main/span_boundary_calibration/tiny_bert_structure_neighborhood_pair_20260429_084508/`。失败 assertion 的输出不作为结果使用，但其原因保留在记录中，因为它促成了训练日志诊断修复。

下一步有边界 PTB/MCTB 诊断被设计成 control/mechanism 匹配对照：PTB train/dev/test sizes 为 `256/64/64`，MCTB 五域各 `64` 句，`structure_neighborhood_loss_scale=0.02`，`structure_neighborhood_loss_margin=1.0`，并启用归一化结构 loss。原计划 BERT-large 运行没有进入训练，因为远端缺少缓存的 `bert-large-uncased` 模型 cache，S3 fallback 又因 `SSLEOFError` 失败。之后 BERT-base GPU fallback 也失败，因为虽然有空闲可见设备，parser 环境中 CUDA 不可用。这些是环境/cache 问题，不是机制失败。

强制 CPU 的 cached BERT-base fallback 完成了同一个匹配诊断。它只作为方向性证据保留，因为它不是已接受的 BERT-large comparator；但它仍能帮助判断是否值得为这个目标继续恢复 BERT-large。训练过程中结构邻域目标一直处于 active 状态，许多 batch 输出了非零 `structure-terms` 和 `structure-positive`。

匹配 BERT-base 诊断指标如下：

| 指标 | Control | Mechanism | Delta |
| --- | --- | --- | --- |
| PTB dev FScore | 28.14 | 27.52 | \-0.62 |
| PTB test FScore | 28.25 | 28.21 | \-0.04 |
| MCTB dialogue FScore | 30.99 | 30.40 | \-0.59 |
| MCTB forum FScore | 27.32 | 27.42 | +0.10 |
| MCTB law FScore | 27.68 | 28.19 | +0.51 |
| MCTB literature FScore | 26.24 | 25.21 | \-1.03 |
| MCTB review FScore | 30.19 | 29.06 | \-1.13 |
| MCTB macro delta | \- | \- | \-0.43 |

这个结果削弱了当前目标。实现没有崩，但信号不够精准：两个域小涨，三个域回退，literature 和 review 的负向变化最大。机制层面的最可能解释是：宽泛 PTB-local 邻域压力学到的是更密的 PTB 结构偏好，却没有隔离真正需要修正的 domain-shift 错误。

#### 为什么选择这条路线

第 027 轮已经关闭 post-decode/global-risk 选择器线：它安全，但不值得晋级。

- score-budget-only global reranking 在 PTB128 上回退 `-0.0556`；
- min-span protection 恢复 `+0.0556` 且没有负例；
- 恢复后的结果与先前选择器持平，说明新 global-risk 特征没有带来额外选择能力。

更底层的结论是：手工 no-gold 阈值不足以解决问题。下一条线应该直接瞄准 parser 对结构邻域学到了什么：父子 span、兄弟节点、同跨度标签、邻近重叠和短 span 抖动。

#### 选择摘要

| 路线 | 决策 | 原因 |
| --- | --- | --- |
| 继续调选择器阈值 | 放弃 | 安全但没有新的选择能力 |
| true n-best / forest reranking | 暂缓 | 当前 source 缺少原生支持；更丰富候选仍需要选择器 |
| 泛化强负例 quota 调参 | 放弃 | 之前的 cap/quota 会压掉有用的 hard-structure 信号 |
| PTB-only 结构邻域一致性 | 选择 | 攻击学到的结构偏好，同时保留比较合同 |

#### 实现计划

1. 检查当前 parser 训练 hook 和之前的 hard-competitor 代码。
2. 添加默认关闭的结构邻域项构造和诊断输出。
3. 运行一棵树 finite-loss smoke。
4. 验证 feature-off 的 compile/import 行为不漂移。
5. 如果合理，运行 tiny BERT save/load/test smoke。
6. 判断 PTB256 有边界对照是否值得做。
7. 在任何更强 BERT-large 放大前，只把有边界对照当作方向性证据。
8. 如果有边界对照混合或负向，就降级原目标，而不是为同一目标追 BERT-large 重跑。

#### 成功标准

- 一棵树 loss 有限且 relation buckets 非空；
- 默认 feature-off 路径不变；
- 诊断包含 relation type、span length、label family、kept/dropped counts 和 loss stats；
- tiny BERT smoke 在任何更大对照前不崩；
- 有边界 PTB/MCTB 诊断在更强放大前不能显著差于 same-data control。

#### 降级标准

- auxiliary loss 非有限或支配主 loss；
- 默认路径漂移；
- 诊断被泛化强负例或浅层标签抖动支配；
- tiny BERT smoke precision/recall 或 save/load/test 行为崩；
- 有边界 PTB/MCTB 诊断在 PTB 持平且 MCTB macro 为负，尤其是多个目标域一起回退。

#### 当前结论

该候选通过了可执行链路门槛，但作为“不改目标继续优化”的路线被降级。它默认关闭、能编译、能产生非空有限的 PTB-only 结构诊断，能完成 tiny Berkeley+BERT train/save/load/test，并且在 PTB256 匹配诊断中确实 active。不过方向性 BERT-base 诊断在 PTB test 中性略负（`-0.04`），MCTB macro 为负（`-0.43`），只有 forum 和 law 改善，dialogue/literature/review 回退。因此不值得为了完全相同目标恢复缺失的 BERT-large cache。

下一步应保留有用教训，而不是保留原目标：结构训练信号必须由更深层失败原因约束，例如不稳定性、歧义性或已知 transfer-relevant 结构错误，而不是把宽泛 local-neighborhood 压力施加到每棵 PTB 树。更强的后续方案应当用 uncertainty/error signatures 来 gate auxiliary loss，或者回到能区分“目标域有益结构变化”和“PTB-specific regularization”的选择机制。

### 阶段 4：按金标跨度筛选可靠强竞争项

#### 当前结论

本轮刚进入实验准备阶段。第028轮结构邻域 loss 已经降级：它能跑通，也确实在训练中触发，但小规模方向诊断显示 PTB test 基本持平、MCTB 五域宏平均约 `-0.43`。因此本轮不继续原样补大模型对照，而是把机制改成“每个金标 span 内只挑少量可靠强竞争项”。

#### 为什么换这个方向

第028轮失败更像是训练信号太散，而不是简单的 loss 系数不合适。广义邻域约束把压力铺到很多局部邻居上，可能没有把模型真正需要学的结构分辨能力凸显出来。

本轮的核心判断是：先按每个金标跨度分组，再挑正 hinge、结构上有解释价值、数量受控的少数竞争项。这样既避免全句全局池化把常见 span 淹没，也避免旧 hard-negative cap 那种把罕见但有价值的竞争项压掉的问题。

#### 当前实验合同

1. 训练仍只使用原始 PTB，MCTB_en 只用于后续评测。
2. 新机制默认关闭，关闭时必须保持 Berkeley+BERT 原路径不变。
3. 第一阶段只做最小烟测：语法/导入检查、单树有限 loss、诊断分桶。
4. 单树诊断健康后，才做 tiny Berkeley+BERT train/save/load/test。
5. tiny 链路通过后，才考虑 PTB256/MCTB64 小规模成对评测。

#### 成功标准

- feature-off 路径不变；
- feature-on 单树 loss 有限且非空；
- 保留的竞争项不是单一的同 span 标签噪声，而能按关系、标签族、跨度宽度和 loss 大小解释；
- tiny Berkeley+BERT 链路不崩；
- 只有上述都过关，才进入跨域小样本方向判断。

#### 降级标准

- loss 非有限或压过主解析目标；
- 默认关闭路径被改动；
- top-k 只是在选通用高 loss 噪声；
- K 太小导致有价值的稀有竞争项被压掉，复现旧 cap 失败；
- tiny 或 PTB/MCTB 小诊断继续平或负，且没有清晰可修的机制原因。
