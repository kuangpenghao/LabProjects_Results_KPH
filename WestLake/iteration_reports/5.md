# 第005主版本：解析机制路线选择

## 结论

本主版本把主线从“改训练数据”转向“优化 Berkeley+BERT 解析机制本身”。这一步吸收了用户关于更深层错误原因的提醒：不能只看 span、标签、边界这些表层指标，还要问这些错误背后的候选空间、解码选择和结构一致性为什么会失败。

## 候选路线

本版本比较了三类方向：

1. 继续做表层后处理或规则校准。
2. 在 CKY / chart 级别保留更丰富的标签或结构候选。
3. 做更颠覆的候选树选择或训练侧结构一致性机制。

## 选择理由

按标签保留 CKY 状态的原型证明：模型内部确实存在可以被更细机制利用的结构信息，不能只在最终 best tree 上做局部修补。但原型本身还不是性能方案，它只是说明机制入口可行。

## 后续影响

本版本之后的实验分两步走：先审计局部标签链和 解析图信号是否能安全转化为最终树收益；如果不能，再转向整树候选池、全局结构风险和训练侧结构一致性。

## 完整阶段记录

本节保留本主版本内的连续实现、验证、问题定位和判断依据。后续同一方向的补充验证继续写入本主版本，只有研究问题、候选机制、比较合同或结论发生实质变化时才新开主版本。

### 阶段 1：解析机制第一性原理转向

#### 1. 本轮结论

本轮正式把主线从“改训练数据”切换到“只用原始 PTB 训练，优化 Berkeley+BERT 解析机制”。

- 最新用户约束明确：标准实验应只在 PTB 本身上训练，不添加更多训练集。
- 质量门控结构数据的五域诊断仍然明显变差，说明继续做合成树或派生树不是当前最可信路线。
- 新主线选择为：span-boundary calibrated parsing，即把“这个 span 是否该成为成分”和“这个成分是什么标签”拆得更清楚，并把 span/boundary 信号加入解码。
- 最新同设置小对照已经跑通：只用原始 PTB 子集、同一个 `bert-large-uncased` 入口，原解析器 test F1 为 1.12，加入边界辅助训练后 test F1 为 2.26。这个数值不能当正式性能，但证明解析机制路线在 Berkeley+BERT 路径里可运行、可保存加载、且值得放大。
- 放大到 64/16/16 的 PTB-only 同设置诊断后，原解析器 test F1 为 3.40，新机制 test F1 为 10.96。它仍然不是正式 MCTB 跨领域成绩，但说明机制信号没有在更大一点的样本上消失。
- 放大到 256/64/64 后，原始“求和式”边界辅助目标退化：control test F1 为 11.87，mechanism test F1 为 9.89。这触发机制降级审查，不能原样继续扩大。
- 修正后采用“按有效 span 数归一化”的边界辅助目标。同一棵 PTB 树上辅助损失从 914.341431 降到 0.746401；同样 256/64/64、同样 Berkeley+BERT 入口下，归一化机制 test F1 为 18.41，高于同批 control 的 11.87。
- 继续放大到 512/128/128 后，归一化机制 test F1 为 26.90，对照原解析器为 14.99。这进一步支持“只改解析机制”的方向，但仍然只是 PTB 子集诊断，下一步必须看 MCTB 五域。
- MCTB 五域诊断已经跑通：同样 512 条原始 PTB 训练条件下，control 五域 macro F1 为 15.68，归一化机制为 26.84；dialogue/forum/law/literature/review 五个领域全部提升。这仍不是正式全量 PTB 成绩，但已经证明“只改解析机制、不加训练数据”有真实跨领域信号。
- 1024/256/256 放大验证也完成：PTB test F1 从 17.64 提到 18.27；MCTB 五域 macro F1 从 18.86 提到 19.35，五个领域仍然全部提升。但相比 512-tree 的 +11.16 macro，这轮只有 +0.49 macro，因此结论必须降级为“稳定但较小的机制收益”，不能再按大幅突破叙述。
- 接下来不应马上换无关思路，也不应继续机械放大同一个辅助目标，而应围绕这个机制分析“能与不能”：它到底改善 boundary、短语组合、附着决策，还是只起到浅层稳定化；确认后再吸收同行的结构归纳、全局约束和解码校准思想继续迭代。

#### 2. 第一性原理分析

句法解析本质上是结构化预测。模型不是只预测一个类别，而是在一句话中选择一组互相兼容的 span，并给这些 span 加标签，最后形成一棵树。

跨领域下降可以从三个基本层面理解：

1. 输入分布变了：论坛、对话、文学和评论的表达方式不同于 PTB 新闻文本。
2. 结构分布变了：同样长度的句子，短语组合、边界位置和嵌套方式可能不同。
3. 输出约束没有变：最后仍然要输出合法 constituency tree。

所以真正值得优化的地方不是“让训练样本变多”，而是“让解析器在边界和结构选择上更稳”。如果解析器过度依赖 PTB 新闻域中常见的 span 长度、局部词汇或标签组合，它在新领域就容易选错边界。一个更合理的机制是让模型先学会稳定判断 span 是否存在，再判断标签是什么。

#### 3. 文献方法的利弊拆解

##### Yang et al. 2022

启发：跨领域 constituency parsing 的下降不能只归因于 OOV，结构差异更关键。 可吸收的精华：评测必须坚持 PTB train -&gt; MCTB_en test，并且领域表现要逐项看。 不能直接照搬的地方：论文主要做挑战分析和基线比较，不直接给出本项目可用的改法。

##### Kitaev and Klein 2018

启发：现代 constituency parser 通过表示和 span scoring 直接影响树结构质量。 可吸收的精华：当前代码里的 fencepost/span chart 机制就是可以动手的位置，改解析器比改数据更贴近问题。 风险：只调整表示或 dropout 可能变成普通调参，必须让机制和 span/boundary 问题直接相关。

##### Kitaev, Cao, and Klein 2018

启发：BERT / pretraining 是强底座，但底座提升不等于解析机制创新。 可吸收的精华：Berkeley+BERT 应作为强基座，机制优化要在这个标准上说话。 风险：如果只是换更强预训练模型，提升不一定能归因于本项目算法。

##### Fried, Kitaev, and Klein 2019

启发：跨域泛化仍然依赖结构化输出预测，预训练有帮助但不是全部。 可吸收的精华：要看解析机制在跨域场景下是否稳定，而不是只看 PTB dev。 风险：小模型诊断不能冒充正式 Berkeley+BERT 可比成绩。

#### 4. 新方法想法

新方法暂定名：PTB-only span-boundary calibrated parsing mechanism。

机制拆解：

1. 训练数据不变：只用原始 PTB train。
2. 模型机制改变：在 span chart 上增加 span-boundary confidence，显式建模“这个 span 是否应成为成分”。
3. 监督来源不变：gold span 和 sampled non-gold span 都来自同一个原始 PTB 句子，不新增训练句子。
4. 目标尺度改变：boundary 辅助损失按有效 span 数归一化，避免长句候选 span 数量把原解析目标压过。
5. 解码改变暂缓：label score 与 boundary confidence 的解码融合单独验证，当前先证明辅助训练路径稳定。
6. 对照清楚：同一批 PTB 句子训练 control 和 mechanism，两者唯一差异是解析机制。

#### 5. 严谨性证明

这个想法合理，不是因为听起来高级，而是因为它满足四个可检查条件。

1. 它正对问题本体。 跨领域错误最终表现为树结构和边界错误，因此优化 span/boundary 决策比继续扩训练数据更贴近根因。

2. 它保持实验口径干净。 没有额外训练句子，没有目标域标签，没有 MCTB_en 参与调参。如果变好，归因更清楚。

3. 它有可实现入口。 当前 Berkeley/Benepar 代码已经有 span label chart、structured loss 和 decode_from_chart，适合插入 boundary score 或校准项。

4. 它可证伪。 如果 feature-on 在同数据、同 seed、同评测链下不如 feature-off，或者只在一个领域偶然提升，就应降级或放弃。

#### 7. 下一步

1. 继续坚持只用原始 PTB 训练，不再回到合成/派生训练树路线；扩大规模时只扩大原始 PTB 子集或最终全量 PTB，不引入额外训练来源。
2. 1024/256/256 的同源 PTB-only Berkeley+BERT 放大验证已经完成，结论是：同数据口径下仍然全域为正，但增益显著缩小。
3. 结构错误分析已经完成：当前机制主要带来中长 span 的召回收益和部分领域结构 F1 提升，但没有解决标签/全局结构一致性问题。
4. 下一版不应只是继续加大 boundary loss；更合理的是吸收结构归纳、全局约束、解码校准等思想，把当前辅助目标迭代成更接近解析运行法则的方法。
5. 暂不尝试无关新思路；只有当更大规模验证失败，或错误分析证明当前机制只带来浅层偶然收益时，再切换机制家族。

#### 7.1 后续分析实验规则

后续每轮主要实验后，都必须尽量补一个分析段，而不是只报告分数。分析段至少回答：

1. 哪些领域或句型改善了，哪些仍然失败；
2. 错误主要来自 boundary、label、短语组合、长距离附着，还是模型过度保守；
3. 这些失败模式如何约束下一轮解析机制设计。

如果当轮资源不足以马上跑完整分析，也要在报告中写清楚分析设计、需要的输出文件、以及它将如何指导下一轮优化。

##### 7.2 1024 棵树 结果后的分析设计

本轮结果说明当前机制不是失败路线，但也不是足够强的最终方法：它在 PTB dev/test 和五个 MCTB 领域都小幅提升，却在训练规模从 512 增到 1024 后明显缩小收益。下一步分析必须回答：

1. 边界层面：机制是否减少了错误 span 的产生，还是只略微提高了已有正确 span 的置信度；
2. 标签层面：剩余错误是否主要来自 span 对了但 label 错了，如果是，单纯 boundary auxiliary 已经不够；
3. 组合层面：长句、嵌套结构和跨域口语/评论句式是否仍然失败，如果是，需要全局结构约束或解码校准；
4. 规模层面：512 到 1024 后收益变小，可能说明原解析器随着 PTB 增多已经学到部分边界规律，当前机制只提供了早期正则化。

分析完成前，不应把下一轮直接定义为“继续加大 boundary loss”。更合理的候选方向是：在 boundary 之外加入 label/span consistency、全局树约束或解码校准，使解析器真正建模成分结构运行法则。

##### 7.3 1024 棵树 结构错误分析结果

这次分析遵守新的分析规则：只用原始 PTB 训练得到的两个模型，固定五个 MCTB_en 领域各 1000 句，不引入任何额外训练数据。分析指标不是官方 EVALB 分数，而是直接比较 gold tree 与 predicted tree 的结构 span 集合，用来回答“新机制到底修复了什么结构错误”。

完整量化报告与可视化已经生成：

- 量化表：`experiments/analysis/ptb1024_structure_errors/structure_error_summary.md`
- 结构化 JSON：`experiments/analysis/ptb1024_structure_errors/structure_error_summary.json`
- 可视化 1：`experiments/analysis/ptb1024_structure_errors/structure_delta_overview.svg`
- 可视化 2：`experiments/analysis/ptb1024_structure_errors/sentence_win_loss_balance.svg`

核心量化结论：

| 领域 | control structural F1 | mechanism structural F1 | delta | 句级 win/loss/tie |
| --- | --- | --- | --- | --- |
| dialogue | 28.79 | 30.10 | +1.31 | 253/189/558 |
| forum | 24.39 | 25.53 | +1.14 | 317/249/434 |
| law | 33.13 | 34.58 | +1.45 | 368/274/358 |
| literature | 22.96 | 23.43 | +0.48 | 267/245/488 |
| review | 28.99 | 31.03 | +2.04 | 267/184/549 |

分解后可以看到，这个机制不是“全面变聪明”，而是有清晰边界：

1. 长度 6-10 与 11+ 的 gold span 召回普遍提升，例如 law 的 11+ span 召回 +5.68，review 的 6-10 span 召回 +4.79，说明 boundary 监督确实帮助了一部分中长结构识别。
2. 非常短的 span 并不稳定：law 的长度 2 span 召回 -1.60，literature 的长度 2 span 召回 -1.92。也就是说，当前机制可能在鼓励更稳的中长边界时，牺牲了一部分局部短语边界。
3. 修复最多的 gold label 主要仍是 VP、NP、PP、S、SBAR；但剩余 missed labels 也集中在 VP、NP、S、PP。这说明 binary boundary objective 只能解决“有没有 span”的一部分问题，不能充分解决“这个 span 应是什么结构角色”。
4. 句级 win/loss/tie 显示提升是稀疏但稳定的：很多句子不变，少部分句子明显改善，也有少部分退化。因此下一轮要做的是减少退化、放大有效结构类型，而不是盲目重复同一个辅助目标。

因此，本轮分析对下一版机制给出三条约束：

1. 保留 PTB-only 训练口径，不回到训练数据改造。
2. 下一版机制必须从 binary boundary 扩展到 label-aware span consistency，让 boundary 判断和短语标签判断互相约束。
3. 解码阶段需要显式保护高置信短 span，同时让中长 span 的 boundary gain 真正转化为合法树结构收益；否则会继续出现“中长 span 变好、短 span 退化”的交换。

##### 7.4 1024 棵树 运行原理级诊断结果

在用户指出 7.3 仍偏“表面现象”之后，我继续做了两组不改数据、不重训模型的机制诊断。目标是回答：当前 gain 到底来自推理时 boundary score 改变了解码，还是训练时辅助目标改变了底层 span/label scorer。

- 原始打分结构化结果：`experiments/analysis/ptb1024_representation_shift/representation_shift_diagnostics.json`
- 原始打分可视化：`experiments/analysis/ptb1024_representation_shift/representation_f1_delta.svg`
- margin shift 可视化：`experiments/analysis/ptb1024_representation_shift/representation_margin_shift.svg`

关键发现 1：当前 checkpoint 中 `span_boundary_decode_scale=0.0`，所以 boundary 分支没有在解码阶段直接给 span 加分。把同一个机制模型分别用 raw decode 与 boundary-on decode 评估，五个领域结果完全一致：

| 领域 | raw structural F1 | boundary-on structural F1 | delta |
| --- | --- | --- | --- |
| dialogue | 30.10 | 30.10 | +0.00 |
| forum | 25.53 | 25.53 | +0.00 |
| law | 34.58 | 34.58 | +0.00 |
| literature | 23.43 | 23.43 | +0.00 |
| review | 31.03 | 31.03 | +0.00 |

这说明当前提升不是“推理时 boundary bonus 修好了结构”，而是训练时 auxiliary boundary objective 对共享表示和 label scorer 的间接塑形。

关键发现 2：control vs mechanism 的原始 chart margin 对比显示，这个辅助目标确实改变了底层打分，但选择性不足：

| span category | count | raw nonempty-margin shift | label-margin shift |
| --- | --- | --- | --- |
| recovered_gold | 4,124 | +0.027 | +0.022 |
| added_error | 23,314 | +0.028 | +0.026 |
| lost_gold | 3,256 | \-0.038 | \-0.028 |
| removed_error | 24,182 | \-0.043 | \-0.027 |
| missed_gold | 32,125 | \-0.006 | +0.000 |
| non_gold_all | 1,424,416 | \-0.010 | \-0.000 |

这里最重要的不是 recovered_gold 被抬高了 +0.027，而是 added_error 也几乎同幅抬高了 +0.028。也就是说，当前 objective 会让一部分正确 span 更容易被选中，但也会把一批错误 span 一起推高。它更像一个弱正则器，而不是能区分“正确结构 vs 竞争错误结构”的解析规则。

因此，下一版方法不应只是继续加大 boundary loss 或简单打开 decode scale。更合理的机制约束是：

1. 加入 label-aware span consistency，让“是否成分”和“是什么结构角色”共同决定，而不是 binary boundary 单独推动。
2. 加入 gold-vs-competing-span contrastive margin，直接压低会与 gold tree 竞争的错误 span，而不是只做全局正负 span 分类。
3. 在解码中保护高置信短 span，并把中长 span 的收益转成合法树结构收益，避免继续出现中长 span 改善但短 span 被牺牲。

#### 8. 当前边界

- 可以改：span scoring、boundary confidence、decode score、loss term、regularization、全局结构约束和解析过程校准。
- 可以扩大：原始 PTB 内部训练子集规模，直到最终 full-PTB 验证。
- 不可以改：训练句子来源、MCTB_en 标签使用、用生成树或派生树补训练。
- 暂不声称：任何 tiny/local 结果达到正式 Berkeley+BERT 可比水平。
- 已降级：原始求和式 boundary 辅助目标；除非有新证据证明尺度问题已被其他方式解决，否则不要原样恢复。

### 阶段 2：文献共性与颠覆性候选

#### 1. 本轮结论

本轮先暂停把“规则一致性小钩子”继续接入训练，改为按用户要求做一次更大的方法复盘。

结论很明确：下一步最值得试的不是继续微调 boundary loss、强负例 数量或简单解码权重，而是把结构约束真正放进解析过程。当前推荐的第一候选是 **label-conditioned rule CKY decoder**：让 CKY 在每个 span 上保留按标签区分的最优状态，并用训练集规则约束父子标签组合，而不是先选好标签再粗略加分。

这个方向比当前小钩子更“底层”，但不是凭空颠覆。它同时满足三个条件：

1. 有直接 MCTB 证据：rule-based decoding 论文在 MCTB 平均 F1 上达到 88.09，高于 Berkeley+BERT 参照 87.20。
2. 不需要使用 MCTB_en 标签训练或调参，仍能保持 PTB -&gt; MCTB_en 的比较口径。
3. 正好解释了当前小钩子的局限：小钩子只是在标签已经选完后给 split 加一点规则分；真正的规则 CKY 是在动态规划里按标签和规则共同选树。

#### 2. 为什么不能继续只做原来的小修小补

过去几轮失败给出的共同信号是：单独增加训练压力，经常会把正确结构和错误结构一起推高，或者用过强的负例压力损害召回。

| 已试方向 | 关键现象 | 对下一步的约束 |
| --- | --- | --- |
| boundary auxiliary | 1024 棵树 MCTB macro 18.86 -&gt; 19.35，提升稳定但很小；recovered gold 和 added error 的 margin 几乎同幅增加 | 不能再只加大 boundary loss；需要选择性结构约束 |
| boundary-label decode fusion | tiny gates 在 collapse、no-effect、低于 control 之间摆动 | 不应继续盲调 decode scale |
| hard competitor / fallback | 修复 fallback 数量后仍低于同设置 control | 不能只管“困难负例数量”，要管负例的结构价值 |
| 结构/词汇数据改造 | 公平诊断系统性不理想，且与当前原始 PTB-only 约束冲突 | 当前不把训练数据改造作为主线 |
| rule-consistency hook | 默认关闭不破坏原结果，打开后合成样例按预期改 split，0.50 -&gt; 1.50 | 说明 decoder 层可动，但当前版本只是近似探针 |

所以这轮的核心判断是：解析器底层职责没有被满足好，尤其是“局部 span/label 分数如何组合成合法且合理的整棵树”这一层。继续只在局部 loss 或局部 bonus 上加力，收益大概率会很有限。

#### 3. 文献共性

##### 3.1 有效方法都让结构成为一等公民

Shi et al. 2022 的 rule-based CKY 不是单纯调参，而是改变解码器状态：每个 span 的最优子树按标签保留，并且只有训练集中出现过的父子规则组合能参与构树。它在 MCTB 上的平均 F1 为 88.09，而 Yang et al. 2022 报告的 Berkeley+BERT 参照为 87.20。

这说明一个重要事实：跨领域结构下降并不一定要靠目标域标签解决，训练域规则仍能帮助跨域，因为常见规则的加权召回在跨域数据里仍然很高。

##### 3.2 LLM 本身不是可靠解析器，但可以做受约束生成器

Li et al. 2023 和 Guo et al. 2025 的共同点不是“让 LLM 直接解析”。相反，它们都承认直接 LLM parsing 有结构合法性和忠实性问题。

真正有效的是：

1. 用 grammar rules 或 masked tree 限制生成；
2. 用 confidence / grammar distance 过滤伪数据；
3. 让生成数据服务于传统 chart parser 或 span representation；
4. 避免无约束生成树污染训练。

这对我们有启发，但也有边界：这些强方法往往会用 target raw sentences 或 LLM-generated target-style data。如果当前仍坚持“只用原始 PTB 训练”，那它们不能直接作为下一步主线，只能作为未来 scope 扩展候选。

##### 3.3 对比学习要按树邻域定义，而不是泛泛地压 强负例

Guo et al. 2025 的 span-level contrastive pretraining 很关键：正例不是随便增强出来的句子，而是 left child、right child、parent、brother；负例也不是泛泛困难负例，而是边界相邻的 invalid spans。

这正好解释我们 hard competitor 路线为什么失败：我们之前更关注“保留多少困难项”和“fallback 是否过多”，但结构价值不够精细。下一版如果做对比学习，必须以真实树邻域为单位，而不是继续调 quota。

##### 3.4 全局目标比独立 span 打分更接近解析本质

Zhang et al. 2020 的 neural CRF parser 说明，树结构的全局归一化目标可以用 GPU inside algorithm 做得足够快；bracketing-then-labeling 也说明“先决定边界，再决定标签”可以变成更完整的架构，而不只是辅助 loss。

这支持一个更宽的判断：如果 rule CKY 原型不稳，下一条备选路线应该是 tree-CRF / bracketing-labeling 级别的架构替换，而不是回到小尺度训练技巧。

#### 4. 候选方向

| 候选 | 改动层级 | 优点 | 风险 | 当前判断 |
| --- | --- | --- | --- | --- |
| A. label-conditioned rule CKY decoder | 解码器 + 训练目标 | 有直接 MCTB 证据；保持 PTB-only；承接当前 rule-hook smoke | 实现比当前小钩子更深，需要处理未知规则和 Cython/图计算复杂度 | 首选 |
| B. structure-neighborhood contrastive pretraining | span 表示 + 训练目标 | 正负例定义更符合树结构，比旧 强负例 更有结构价值 | 最强版本可能需要 target raw 或生成数据；PTB-only 版本收益不确定 | 第二候选 |
| C. tree-CRF / bracketing-then-labeling replacement | 全局目标 + 解析架构 | 从根上处理树分布，而不是局部 span 独立打分 | 工程面更大，离当前代码更远 | 备选架构 |
| D. direct LLM parser | 外部解析器替换 | 形式上最颠覆 | 文献显示树合法性和忠实性不足 | 暂时拒绝 |

#### 5. 推荐先试 A 的原因

我推荐先试 A，不是因为它最保守，而是因为它的“颠覆程度”和“证据强度”最平衡。

1. 它直接替换 Berkeley/BERT 的关键底层职责：不是只让 scorer 多一个分数，而是让 decoder 重新决定“什么标签的子树可以组合成什么父节点”。
2. 它有最直接的跨领域证据：MCTB 平均 88.09，相比 Berkeley+BERT 87.20 有明确提升。
3. 它不要求改变训练数据来源，因此不会混淆当前比较口径。
4. 我们当前已经有 rule table 与 conservative hook 的工程入口，可以先做 standalone prototype，不必一上来冒险重写整套训练。

#### 6. 下一步执行计划

下一步不直接上 3090 长训练，而是先做三道低成本门槛：

1. 写一个独立的 label-conditioned CKY 原型，用合成 chart 和少量 PTB-derived rule table 验证算法行为。
2. 检查未知规则和规则覆盖：不能只看合成样例是否变好，还要看真实 PTB chart 是否会被规则表过度阻塞。
3. 通过后再接入 Berkeley/BERT parser flags，做 tiny 同数据 control/mechanism 对照。

通过标准：

- feature-off 路径必须完全等价于原 parser；
- rule-based decode 不能产生空树、全阻塞或 collapse；
- tiny gate 至少要稳定，不应复现 boundary decode 那种 scale-sensitive 崩溃；
- 若 tiny gate 无效，再转到 B 或 C，而不是继续微调当前小钩子。

#### 7. 本轮保留和降级

保留：

- 当前 rule-consistency diagnostic 和 rule-hook smoke，作为后续 label-conditioned CKY 的工程证据。
- hard-competitor 日志与质量门控工具，作为未来 structure-neighborhood contrastive 的诊断材料。
- boundary auxiliary 分析结果，作为“非选择性 span margin 会失效”的反例。

降级：

- 直接把当前 approximate rule hook 接入训练：暂缓。它太像一个低成本探针，还不是最有文献支撑的最终方向。
- 继续 boundary scale tuning：拒绝。
- 直接 LLM parsing：拒绝。
- 在当前严格 PTB-only 口径下直接引入 target-domain generated treebank：暂缓，除非以后明确扩展训练数据范围。

#### 8. 对用户问题的直接回应

是的，Berkeley/BERT 应该被拆成模块来审：encoder 给表示，span scorer 给局部分数，decoder 负责把局部分数组成树，training objective 决定模型学什么结构差异。

现在最像底层瓶颈的是 decoder 和 objective 的组合：现有 parser 太依赖独立 span/label 分数，而跨领域失败恰恰发生在这些局部分数如何组合成合法结构树的地方。因此下一轮应该做真正的结构约束解析器，而不是继续在表层训练技巧上试错。

### 阶段 3：按标签规则 CKY 原型烟测

#### 结论

这轮原型证明了核心机制可行：按标签保留 CKY 状态后，解码器可以在全局树结构里使用父子标签规则，而不是只在局部跨度打完分后再拼树。

但下一步不应该直接做硬规则接入。PTB 规则覆盖随表规模快速升高，1024 棵表树已经达到 98.6089% 二元规则覆盖；不过仍有未知规则，所以更合理的下一步是软规则加分/惩罚，而不是把未知规则直接禁止。

#### 机制验证

合成样例里，普通 CKY 和无约束规则 CKY 分数完全一致：

- 普通 CKY：12.000000
- 无约束规则 CKY：12.000000

加入规则约束后，解码器改选了局部分数更低但规则合法的树：

- 规则 CKY：10.000000
- 普通树未知规则数：2
- 规则树未知规则数：0

这说明它确实是在做“全局结构约束下的动态规划”，不是简单的后处理补丁。

软规则版本也通过了安全阀测试：

- 轻惩罚：11.000000，仍会选择带 2 条未知规则的局部强证据树
- 强惩罚：10.000000，改选 0 条未知规则的合法树

这说明未知规则不是被彻底禁止，而是可以通过惩罚强度控制；这对跨领域更安全。

#### 覆盖风险

| 规则表 / 评估树 | 覆盖率 |
| --- | --- |
| 8 / 8 | 68.8679% |
| 64 / 64 | 86.5465% |
| 256 / 256 | 94.9870% |
| 1024 / 1024 | 98.6089% |

覆盖结果支持继续，但不支持硬卡死。跨领域文本中可能出现少量 PTB 不常见但合法的结构，所以未知规则应被惩罚或平滑，而不是直接不可选。
